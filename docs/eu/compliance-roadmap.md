#### Article 12: Record-Keeping

```typescript
// AI Act Audit Log Schema
interface AIAuditLog {
  // Identification
  logId: string;                    // UUID v4
  systemId: string;                 // AI System identifier
  systemVersion: string;            // Semantic version
  
  // Temporal
  timestamp: ISO8601;               // UTC timestamp
  retentionUntil: ISO8601;          // 6 years from timestamp
  
  // Event Details
  eventType: 'DECISION' | 'OVERRIDE' | 'INCIDENT' | 'MAINTENANCE';
  inputData: EncryptedJSON;         // Input to AI system
  outputData: EncryptedJSON;        // Output from AI system
  confidence: number;               // Model confidence score
  
  // Governance
  riskLevel: 'LOW' | 'MEDIUM' | 'HIGH' | 'CRITICAL';
  humanOversight: boolean;          // Was human involved?
  humanReviewer?: string;           // User ID if oversight applied
  
  // Technical
  modelHash: string;                // SHA-256 of model binary
  featureImportance: JSON;          // XAI explainability data
  
  // Compliance
  gdprConsent: boolean;             // Valid consent on record
  dataRetentionValid: boolean;      // Within retention policy
}
```

**Retention Requirements:**
- Minimum: 6 years from AI system deployment (Article 12(1))
- Format: Immutable, tamper-evident storage
- Access: Authorized personnel only, audit trail of access

### 4.2 CE Marking for AI Systems

```
CE MARKING PROCESS
═══════════════════════════════════════════════════════════════════════════════════════════════

Step 1: Conformity Assessment Procedure Selection
├─ High-risk AI → Module B (EU-type examination) + Module D (Quality assurance)
└─ Selected: Notified Body involvement required

Step 2: Technical Documentation Preparation
├─ System description and intended purpose
├─ Risk assessment and mitigation measures
├─ Data governance and quality management
├─ Transparency and user information
├─ Human oversight measures
└─ Accuracy, robustness, cybersecurity measures

Step 3: Notified Body Engagement
├─ Select notified body (TÜV SÜD, BSI, DEKRA, Bureau Veritas)
├─ Submit technical documentation
├─ Undergo assessment audit
└─ Receive EU-type examination certificate

Step 4: Quality Assurance Implementation
├─ Implement quality management system
├─ Establish production control procedures
├─ Conduct internal audits
└─ Document conformity

Step 5: CE Marking Application
├─ Draft EU Declaration of Conformity
├─ Affix CE marking to AI system
├─ Register in EU database (per Article 71)
└─ Publish transparency information

═══════════════════════════════════════════════════════════════════════════════════════════════
```
