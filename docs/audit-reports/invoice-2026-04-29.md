## Root Cause Analysis (BLOCCANTI)

### F-001/F-002/F-003/F-004: Prisma update() senza tenantId

```typescript
// PRIMA (VULN — qualsiasi tenantId poteva aggiornare):
await this.prisma.invoice.update({
  where: { id: invoiceId },
  data: { status: 'PAID' },
});

// DOPO (SAFE — update isolato per tenant):
await this.prisma.invoice.updateMany({
  where: { id: invoiceId, tenantId },
  data: { status: 'PAID' },
});
```

**Impatto:** Cross-tenant write — tenant A poteva marcare come PAID fatture di tenant B.
**Traceability:** OWASP A01:2025 (Broken Access Control), GDPR Art.32 (Integrity)

### F-005: handlePaymentWebhook senza tenantId

```typescript
// PRIMA (VULN):
async handlePaymentWebhook(invoiceId, stripeSessionId, stripeEventId)

// DOPO (SAFE):
async handlePaymentWebhook(invoiceId, tenantId, stripeSessionId, stripeEventId)
```

**Impatto:** Webhook Stripe poteva marcare come PAID fatture di tenant sbagliato.
**Traceability:** PCI DSS v4.x §3.5
