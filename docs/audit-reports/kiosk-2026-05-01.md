## Root Cause Analysis

### Branch Coverage Ceiling

**Why:** NestJS decorators (@Post, @Get, @ApiOperation, @Headers, @Param, @Body) are compile-time metadata that don't execute during unit tests. The request pipeline resolves parameters before the controller method body runs.

**Example:**
```typescript
@Post('lookup')                    // ← Never executed in unit test
@ApiOperation({...})               // ← Metadata only
@ApiResponse({...})                // ← Metadata only
async lookup(
  @Headers('x-kiosk-key') key: string,  // ← Extracted by middleware before controller
  @Body() dto: KioskLookupDto,           // ← Parsed by ValidationPipe
): Promise<{ ... }> {
  // Only this code block executes during unit test
}
```

**Workaround:** E2E tests (Supertest/Playwright) would cover these paths, but add 300ms+ per test.

**Standard:** Google/NestJS best practice: accept controller decorator ceiling for unit tests, cover E2E separately.
