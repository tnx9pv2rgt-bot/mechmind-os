## Root Cause Analysis (BLOCCANTI)

### F-001: SMS body in plaintext — GDPR Art.32 violation

```typescript
// PRIMA (VULN): body salvato raw
const message = await this.prisma.smsMessage.create({
  data: { threadId, direction: SmsDirection.OUTBOUND, body, status: SmsStatus.SENT }
});

// DOPO (SAFE): encrypt-on-write, decrypt-on-read
// costruttore aggiornato:
constructor(
  private readonly prisma: PrismaService,
  private readonly encryption: EncryptionService, // aggiunto
) {}

// write (sendMessage e receiveInbound):
body: this.encryption.encrypt(body)

// read (getMessages):
const decryptedMessages = messages.map(m => ({
  ...m,
  body: this.encryption.decrypt(m.body),
}));

// read (receiveInbound idempotency return):
return { ...existing, body: this.encryption.decrypt(existing.body) };
```

**Impatto:** PII (contenuto SMS clienti) era leggibile in plaintext. Violazione diretta GDPR Art.32.
**Traceability:** GDPR Art.32, OWASP A02:2021 Cryptographic Failures, CLAUDE.md PII Rule
