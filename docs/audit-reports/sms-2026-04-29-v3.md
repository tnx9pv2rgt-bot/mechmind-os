## Root Cause Analysis — BLOCCANTE RISOLTO

### F-001: SMS body in chiaro — GDPR Art. 32 violation

```typescript
// PRIMA (VULN): corpo SMS salvato in plaintext
const message = await this.prisma.smsMessage.create({
  data: { threadId, direction: SmsDirection.OUTBOUND, body, status: SmsStatus.SENT },
});
// body = "Appuntamento alle 10:00 per il cambio gomme" → DB in chiaro

// DOPO (SAFE): encrypt-on-write, decrypt-on-read
constructor(
  private readonly prisma: PrismaService,
  private readonly encryption: EncryptionService,
) {}

// Write:
body: this.encryption.encrypt(body),  // AES-256-CBC via EncryptionService

// Read (getMessages):
const decryptedMessages = messages.map((m) => ({
  ...m,
  body: this.encryption.decrypt(m.body),
}));

// Idempotency return (receiveInbound):
return { ...existing, body: this.encryption.decrypt(existing.body) };
```

**Impatto risolto:** Compliance GDPR Art. 32 ristabilita. PII (corpo SMS) ora cifrato at-rest con AES-256-CBC via chiave da env `ENCRYPTION_KEY`. Test aggiornati con pattern identity-mock per verificare le chiamate encrypt/decrypt senza alterare i valori assertion.
