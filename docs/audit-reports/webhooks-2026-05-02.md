## CRITICAL FINDINGS

### BUG-001: TENANT ISOLATION MISSING (BLOCCANTE)

**Severity:** 🔴 BLOCCANTE  
**OWASP:** A01:2025 Broken Access Control  
**GDPR:** Article 32 Encryption & Access Control  

**Description:**
All 5 webhook endpoints accept payloads **without tenantId filtering**:
- `POST /webhooks/segment`
- `POST /webhooks/zapier`
- `POST /webhooks/slack/events`
- `POST /webhooks/slack/commands`
- `POST /webhooks/crm/:provider`

**Impact:**
If module is activated in app.module.ts, external integrations (Segment, Zapier, Slack, CRM systems) can send data on behalf of ANY tenant without isolation checks. This violates GDPR Article 32 (access control requirements) and creates cross-tenant data leakage.

**Example Attack:**
```
POST /webhooks/segment
{
  "type": "track",
  "userId": "attacker-tenant-user",  // Can spoof any userId
  "event": "Booking Created",
  "properties": { "bookingId": "victim-tenant-123" }  // Cross-tenant leak
}
```

**Required Fix:**
1. Add `@Query() tenantId: string` parameter to all 5 endpoints
2. Verify tenantId matches authenticated tenant context (via JWT or request header)
3. Pass tenantId to service handlers for audit logging
4. Document in API spec: "All webhook endpoints require X-Tenant-Id header"

**Status:** MUST_FIX_BEFORE_MERGE (module remains orphan until fixed)
