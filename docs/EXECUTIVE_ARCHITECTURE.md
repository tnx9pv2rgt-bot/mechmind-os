# рҹҸ—пёҸ SYSTEM ARCHITECTURE

## Architecture Principles

| # | Principle | Description | Implementation |
|---|-----------|-------------|----------------|
| 1 | **рҹ”’ Security by Design** | Security is not an add-on but foundational | Zero-trust network, encryption at rest/transit, least privilege |
| 2 | **рҹҸў Tenant Isolation** | Strict data segregation between tenants | Row-level security, schema-per-tenant optional, tenant context propagation |
| 3 | **вҡЎ Event-Driven** | Loose coupling through async events | NestJS EventEmitter, webhook system, event sourcing for audits |
| 4 | **рҹ’° Cost Optimization** | Efficient resource utilization | Connection pooling, Redis caching, selective indexing |
| 5 | **рҹ‘ҒпёҸ Observability** | Full system visibility | Structured logging, distributed tracing, metrics aggregation |

**Flow Description:**
1. System event triggers notification (booking created, WO completed, etc.)
2. Event bus routes to Notification Module
3. Template engine renders message with Handlebars
4. Channel selector chooses delivery method based on user preference
5. External APIs called (Resend for email, Twilio for SMS)
6. Delivery status tracked in Redis
7. Full audit trail logged to PostgreSQL
